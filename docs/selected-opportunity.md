# Selected Semester Project Opportunity

## Selected Project Title
SmartCampus Centralized Lost-and-Found Tracking System

## Target User
University students who lose personal belongings, campus security guards managing physical holding bins, and student club administrative leads.

## Problem Statement
University students frequently lose vital items (keys, IDs, tablets) on campus and suffer significant time delays because tracking alerts are scattered across disjointed social media chats, causing items to sit uncollected in physical lockups.

## Current Alternative
Students post unstructured messages in large chat groups (LINE/Discord), search physical security hubs manually, or post text descriptions on notice boards.

## Proposed IT Solution
A centralized web-based database directory where finder users submit item logs with image attachments via a structural form, and loser users search filtered category tables (e.g., Electronics, Wallets) sorted by time and location.

## Why This Opportunity Was Selected
This venture achieved our highest NUF ranking (Score: 14/15). It tackles a highly specific, repetitive daily campus problem while presenting exceptionally strong technical feasibility for a 3-member team within 14 weeks.

## Why This Is Feasible Without IoT or Advanced Cybersecurity
The system operates exclusively on user-driven web inputs and relational tables. It removes all needs for physical proximity sensors, Bluetooth tags, or RFID asset tags. Data submission is authenticated via a simple web form data schema, avoiding advanced cryptography or penetration testing scopes.

## Possible MVP Type
Landing Page Form-Based MVP integrated with an Airtable item search directory.

## Initial Validation Plan for Lab 03
* **Who we will ask:** We will survey at least 15 on-campus undergraduate students and interview 2 members of university facility/security staff.
* **What evidence we will collect:** We will gather empirical data on loss frequency per semester, time spent searching for items, and the success rate of the current LINE group workaround.
* **What question must be answered first:** Will students actively open a dedicated digital dashboard tool rather than defaulting to their daily chat apps to report a found item?
