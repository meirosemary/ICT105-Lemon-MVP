# IT Venture Ideas Analysis

## Idea 1: Smart Campus Lost-and-Found System
* **Problem Area:** Students lack a centralized, reliable system to report or search for lost items on campus.
* **Target User:** University students, security staff, and university administrative staff.
* **Current Alternative:** Informal group chats (LINE, Messenger) or visiting security offices in person.
* **Possible IT Solution:** A web-based platform featuring item categorization, search/filtering, image upload, and a messaging portal.
* **Semester Feasibility:** Highly Feasible. Uses standard CRUD operations and basic database filtering.

---

## Idea 2: Digital Notice Board
* **Problem Area:** University event information is scattered across different departments, leading to missed deadlines.
* **Target User:** University students and student event organizers.
* **Current Alternative:** LINE group chats and physical notice boards.
* **Possible IT Solution:** An event catalog web dashboard with registration deadlines and automated notification alerts.
* **Semester Feasibility:** Feasible. Easily achievable if SMS alerts are replaced with free web/email notifications.

---

## Idea 3: Student Skill-Share and Project Matchmaker
* **Problem Area:** Students struggle to find peers outside their friend groups who possess complementary technical or design skills for projects.
* **Target User:** University students seeking project collaborators or startup team members.
* **Current Alternative:** Random professor grouping or unstructured social media forum posts.
* **Possible IT Solution:** A profiling dashboard matching user-tagged skills with project requirements via database filtering.
* **Semester Feasibility:** Highly Feasible. Strong focus on relational database queries and UI/UX profile cards.

---

## Idea 4: Campus Navigation (Rangsit University)
* **Problem Area:** Navigating a large campus like Rangsit University to find specific classrooms, facilities, or offices is confusing for newcomers.
* **Target User:** New/current students, visitors, parents, and university staff.
* **Current Alternative:** Static PDF campus maps or asking bystanders for directions.
* **Possible IT Solution:** A web-based interactive map and directory dashboard using custom overlays or the Google Maps API.
* **Semester Feasibility:** Moderately Feasible. Highly viable if scoped as a searchable building/room directory rather than turn-by-turn navigation.
